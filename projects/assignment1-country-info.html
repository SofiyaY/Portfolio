A small REST web application in Go that aggregates country data from external APIs and exposes a clean client-facing interface for country info, population history, and service diagnostics.

Overview

Goal: provide a simple facade over two course-hosted services (CountriesNow, REST Countries) and unify their outputs into one consistent schema. I focused on concise endpoints, predictable JSON structures, and graceful error handling. Cities are listed alphabetically and can be limited via a query parameter to keep responses fast.

Architecture

  • Routing: gorilla/mux with three handlers: info, population, status.
  • Info: merges core country attributes and capital + cities listing (ascending order).
  • Population: filters years via limit=start-end, computes mean (rounded integer).
  • Status: returns upstream HTTP codes and uptime since service start.
  • UX: helpful guidance messages when parameters are missing or invalid.

Endpoints

  • GET /countryinfo/v1/info/{code}{?limit=10} â€” general country info (name, continents, population, languages, borders, flag, capital, cities)
  • GET /countryinfo/v1/population/{code}{?limit=start-end} â€” population values per year and mean
  • GET /countryinfo/v1/status â€” dependency status and uptime

Implementation

  • Routing with gorilla/mux
  • External APIs: CountriesNow and REST Countries (course-hosted instances)
  • Graceful error handling and input validation
  • Alphabetical city listing with optional limit

  • Limiting city lists and population ranges improved responsiveness.
  • Clear errors helped debugging during rate limits or upstream hiccups.
