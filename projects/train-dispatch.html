Highlights

  • Layered design: domain (TrainDeparture), register, printer, utility
  • OOP principles: encapsulation, validation, modular responsibilities
  • Unit tests (JUnit) for core behaviors
  • Maven build, simple CLI/UI for interaction
  • Streams and basic functional programming elements

What I learned

  • Structured, well-documented code aligned with good design principles (coupling/cohesion).
  • Applying OOP concepts (classes, objects, encapsulation) to model a real domain.
  • Working with collections, control structures, and collaboration between objects.
  • Basic exception handling and writing robust code.
  • Unit testing with JUnit and using an IDE/debugger effectively.
  • Intro to functional programming elements (lambdas/streams) with collections.

Improvements

  • Persist schedules to a lightweight database (SQLite) instead of in-memory for durability.
  • Strengthen validation with richer domain rules (track conflicts, delay propagation).
  • Expand unit tests and add integration tests around register/print flows.
  • Refactor CLI into a minimal GUI or web interface for clearer user interactions.
