Group assignment focused on designing and implementing a relational database for a fitness tracker scenario, and building a Streamlit interface to demonstrate CRUD operations and a visualization. We practiced end-to-end data modeling and hands-on SQL with MySQL/SQLite.

Overview

The deliverables included:

  • ER/EER diagram and data dictionary for core entities.
  • Logical schema with primary/foreign keys and integrity constraints.
  • Normalization of all tables (1NF, 2NF, 3NF) to reduce redundancy.
  • Streamlit CRUD demo over selected tables (Create, Read, Update, Delete).
  • Visualization using Plotly based on a multi-table SQL query (joins/aggregates).

Database design

  • Entities & relations: users, activities/sessions, devices, metrics; clear PK/FK structure.
  • Integrity: constraints for referential integrity and sane defaults.
  • Normalization: refactored attributes to achieve 3NF; avoided duplication and anomalies.
  • Implementation: MySQL and SQLite, with sample data seeded for the demo.

Streamlit CRUD

  • Read: interactive tables for browsing records.
  • Create: input forms and validation for inserts.
  • Update: editable forms to modify records safely.
  • Delete: guarded deletions with confirmations.
  • Security: basic measures around input validation and safe SQL operations.

Visualization

  • Complex SQL query (joins/aggregates) distilled into a Plotly chart to surface useful business insights.
  • Clear caption summarizing findings and how they inform decision-making.

What I learned

  • Data modeling (ER/EER) and translating models into robust relational schemas.
  • Practical SQL across MySQL/SQLite, including constraints, indexing basics, and query optimization.
  • Building simple, useful UIs with Streamlit to exercise CRUD and visualizations.
  • Ensuring data integrity and reducing redundancy via normalization.

Improvements

  • Expand indexing strategy and profiling for performance on larger datasets.
  • Strengthen constraints, migrations, and test coverage across CRUD flows.
  • Add auth and roles to Streamlit app for safer multi-user scenarios.
